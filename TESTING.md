# خطة اختبار Ebaa Visits

هذا الملف مخصص لجولة QA اليدوية قبل النشر. نفّذ الهجرات أولًا، ثم شغّل بيانات الاختبار من `supabase/seed.sql`.

## حسابات الاختبار

كلمة المرور لكل الحسابات:

```text
EbaaTest123!
```

| الدور | البريد الإلكتروني | الاسم |
| --- | --- | --- |
| مسؤول النظام | `admin@ebaa.local` | نورة السعدي |
| مدير عام | `manager@ebaa.local` | عبدالعزيز القحطاني |
| مشرف فرع جدة | `jeddah.supervisor@ebaa.local` | هند الحربي |
| مشرف فرع الرياض | `riyadh.supervisor@ebaa.local` | خالد العتيبي |
| موظف مبيعات جدة | `sales.jeddah@ebaa.local` | محمد الغامدي |
| موظف مبيعات الرياض | `sales.riyadh@ebaa.local` | سارة الدوسري |
| موظف مبيعات الدمام | `sales.dammam@ebaa.local` | فيصل الشهري |
| موظف مبيعات مكة | `sales.makkah@ebaa.local` | ريم القرشي |
| مستخدم غير نشط | `inactive@ebaa.local` | تركي السبيعي |

## تجهيز البيئة

- [ ] تطبيق الهجرات بالترتيب من `supabase/migrations`.
- [ ] تشغيل `supabase/seed.sql` من Supabase SQL Editor أو عبر CLI.
- [ ] ضبط `NEXT_PUBLIC_SUPABASE_URL`.
- [ ] ضبط `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
- [ ] ضبط `SUPABASE_SERVICE_ROLE_KEY` للخادم فقط.
- [ ] تشغيل التطبيق محليًا عبر `npm run dev -- -p 3001`.
- [ ] فتح `/login`.

## صلاحيات RLS

- [ ] موظف مبيعات جدة لا يرى عملاء موظفي الرياض أو الدمام أو مكة.
- [ ] موظف مبيعات جدة لا يرى زيارات موظفين آخرين.
- [ ] موظف مبيعات جدة لا يرى متابعات موظفين آخرين.
- [ ] مشرف فرع جدة يرى بيانات فرع جدة فقط.
- [ ] مشرف فرع جدة لا يرى بيانات فرع الرياض.
- [ ] المدير العام يرى بيانات كل الفروع في العملاء والزيارات والتقارير.
- [ ] مسؤول النظام يستطيع إدارة الإعدادات.
- [ ] موظف المبيعات لا يستطيع فتح `/settings`.
- [ ] المستخدم العام في `/qr/[branchId]` لا يستطيع قراءة أي بيانات داخلية.
